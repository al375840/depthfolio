# Knowledge skills

Concentrated, reusable knowledge bundles. Each `SKILL.md` is a self-contained reference for a single technical domain, written for the future-you who has forgotten the specifics and wants to ramp back up fast.

Each skill answers, for its domain: **what is it, why this choice, the gotchas, and the minimum-viable code to be productive**.

| Skill | Topic |
|---|---|
| [browser-ml](./browser-ml/SKILL.md) | Running ONNX models in the browser with WebGPU |
| [three-displacement](./three-displacement/SKILL.md) | Fake 3D from a depth map with Three.js displacement plane |
| [angular-feature-architecture](./angular-feature-architecture/SKILL.md) | Modern Angular layout: standalone, signals, features-based |
| [azure-functions-cleanarch](./azure-functions-cleanarch/SKILL.md) | Clean Architecture in a .NET 8 Azure Function |
| [azure-static-web-apps-deploy](./azure-static-web-apps-deploy/SKILL.md) | Azure SWA deployment, configuration and gotchas |

## Conventions

- Each skill is portable — copy the folder elsewhere and it still makes sense.
- Code blocks are minimum viable, not exhaustive. They show the shape, not the entire implementation.
- Gotchas section is non-negotiable. Every domain has them; writing them down is the point of the skill.
